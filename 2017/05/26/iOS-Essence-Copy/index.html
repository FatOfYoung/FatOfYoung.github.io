iOS深复制和浅复制-再研究

发表于 2017-05-26 | 分类于 iOSDe | | 阅读次数

无论是集合类对象，还是非集合类对象，接收到copy和mutableCopy消息时，都遵循以下准则：

  • copy返回immutable对象；所以，如果对copy返回值使用mutable对象接口就会crash；
  • mutableCopy返回mutable对象；

非集合类对象的copy与mutableCopy

系统非集合类对象指的是NSString，NSNumber等对象。

immutable对象的copy

通过查看打印的信息，str,strC1,strM1均指向同一个地址，进行了指针拷贝(ShallowCopy)。而strC2和strM2开辟了新的地址，进行了内容拷贝(DeepCopy)。

mutable对象的copy

strC1,strC2,strM1,strM2的地址均与str不同，都为内容拷贝(DeepCopy)。而且此时申明为mutable对象的strC2会变成__NSCFConstantString不可变对象，申明为immutable的strM1成为了__NSCFString可变对象。

结论

  • 无论对于集合或非集合类immutable对象进行copy时，是指针复制，mutableCopy时则为内容复制，即浅拷贝。
  • 无论对于集合或非集合类mutable对象，进行copy或mutableCopy均为内容复制，即深拷贝。
  • 在进行copy或mutableCopy时，集合类中的元素均为指针拷贝。
